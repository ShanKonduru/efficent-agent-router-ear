# EAR v0.10.0 Release Notes (Draft)

> Superseded for current delivery status by `docs/releases/v0.10.1-release-notes-draft.md` (current draft title: EAR v0.10.3 Release Notes).

## Summary

v0.10.0 finalizes a major CI/CD hardening cycle completed between v0.1.0 and v0.10.0. The release train focused on deterministic test reliability, strict quality gates, secure publishing, and release automation resilience.

## Highlights

- 100% test coverage gates enforced for publish workflows.
- Predictive Routing Engine delivered with full branch coverage.
- Stable TestPyPI and PyPI publishing workflows with OIDC support.
- Security automation pipeline improved with pip-audit and trivy integration.
- Release/tag verification logic expanded to support both `X.Y.Z` and `vX.Y.Z` tags.

## Notable Changes Since Early Baseline

### Routing Engine Delivery

- Implemented Predictive Routing Engine (E3) with deterministic intent classification.
- Added scoring and candidate ranking behavior with tie-break handling.
- Added broad test coverage for classifier, scoring, filtering, ranking, and decision output.

### CI and Quality Gates

- Added strict test gate before package publication.
- Enforced `--cov-fail-under=100` in release pipelines.
- Improved CI stability around warning-driven test failures.

### Packaging and Publishing

- Aligned distribution metadata and workflow guardrails for reliable release.
- Improved tag/version verification behavior and release workflow gating.
- Fixed publish pipeline wiring issues where publish steps could be skipped.

### Security Automation

- Added and stabilized pip-audit and trivy workflows.
- Introduced temporary pip-audit CVE exception for `CVE-2026-3219` to keep CI actionable while upstream fix status is pending.
