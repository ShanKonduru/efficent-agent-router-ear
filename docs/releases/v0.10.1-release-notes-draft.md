# EAR v0.10.2 Release Notes

> Note: Tag `0.10.1` was created 2026-05-01 against an earlier commit. This release captures all work delivered after that tag, including full E4–E9 implementation.

## Summary

v0.10.1 completes the EAR delivery track for E1-E9 with strict coverage enforcement, validated forced-failure CI behavior, and security-report artifact validation.

## Highlights

- E4 CLI implemented with `route`, `inspect-models`, and `stats` commands.
- E6 Guardrails implemented for prompt-injection and PII detection with vetted-provider filtering.
- E7 Metrics implemented with session-level aggregation and reset behavior.
- E8 MCP server implemented with `route_and_execute` tool and `ear://session/stats` resource.
- CI hardened with `--cov-fail-under=100`, validated forced-failure behavior, and HTML report existence checks for pip-audit and trivy workflows.
- Temporary pip-audit CVE ignore (`CVE-2026-3219`) removed from workflow gates after verification run returned clean.
- `master` and `main` branch heads synchronized.
